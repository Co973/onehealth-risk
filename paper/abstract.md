# Abstract

One Health surveillance projects often depend on sensitive clinical and site data that cannot be uploaded to shared services or redistributed with public software artifacts. We present One Health Risk, a local-first and security-aware Python toolkit for reproducible One Health surveillance risk modelling. The toolkit provides a command-line workflow for schema validation, configurable clinical/environmental/animal/geographic feature assembly, baseline model training, local calibration, prediction export, model cards, reproducibility reports, and local security audits. A synthetic demo dataset allows reviewers to reproduce the complete workflow without private patient records. A Semarang leptospirosis severity pipeline is included as a local/private feasibility demonstration, not as a clinically validated diagnostic model. The contribution is reproducible infrastructure for sites with non-shareable data, with security-aware defaults that reduce accidental disclosure risk while preserving local governance responsibility.
